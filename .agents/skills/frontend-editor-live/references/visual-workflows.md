# Visual workflows

## Layout e responsive

1. Leggere parent, ordine, layout e stili calcolati.
2. Catturare canvas prima della modifica.
3. Preferire ordine/Flex/Grid; usare margini o coordinate solo se il contenitore lo richiede.
4. Applicare la modifica alla revisione corrente.
5. Validare e confrontare desktop, tablet e mobile; controllare overflow, sovrapposizioni, focus e contrasto.
6. Catturare preview aggiornata e offrire undo.

## Comportamento e dati

1. Leggere evento, flow, input/lista vicini, binding, schema e sorgenti.
2. Chiedere conferma prima di creare una sorgente o servizio mancante.
3. Creare validazione, operazione, success/error e aggiornamento UI nella stessa transazione.
4. Provare input valido e invalido in preview; verificare loading, empty ed error state.
5. Controllare log runtime, persistenza e undo.
