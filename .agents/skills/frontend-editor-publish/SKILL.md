# Frontend Editor Publish

1. Validate the visual graph and capability requirements first.
2. Preserve imported original files and never overwrite an external folder without confirmation.
3. Configure the target through `set_export_config`.
4. Request export/build through the editor, then run the produced artifact independently.
5. Compare editor, preview and export visually and functionally.

Use `$frontend-editor-live`; external credentials, store publication and production signing require user authority. Read [references/publish-tools.md](references/publish-tools.md).
