# Kyro App

1. Read the indexed project, pages and current selection with `$kyro-live-context`.
2. Translate the requested product into the smallest coherent set of pages, navigation and app settings.
3. Reuse components and flows rather than duplicating them.
4. Keep beginner-facing defaults safe: responsive layout, visible states, keyboard access, offline behavior and platform fallbacks.
5. Delegate details to `$kyro-design`, `$kyro-actions`, `$kyro-data` and `$kyro-native` while keeping one approved operation plan.
6. Validate routes, graph references and the target export after applying.
