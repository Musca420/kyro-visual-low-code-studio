# Kyro Publish

1. Inspect export target, project capabilities and validation errors with `$kyro-live-context`.
2. Configure export through typed app/export operations; do not patch generated output as the source of truth.
3. Resolve native permissions and dependencies through `$kyro-native` and `$kyro-extensions` before building.
4. Generate readable open code and compare editor, preview and independent runtime.
5. Report build commands, output paths and observed limitations. Signing, stores, paid services and production credentials require user authority.
