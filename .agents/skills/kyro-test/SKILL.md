# Kyro Test

1. Validate the graph and capture the current preview through the Kyro MCP.
2. Exercise the requested success, validation, empty, loading, error and undo paths through visible controls.
3. Check desktop, tablet and mobile layouts plus keyboard focus order and accessible names.
4. Save, close and reopen when persistence is in scope.
5. For publish work, run the independent export and compare behavior with preview.
6. Record reproducible evidence. Never report a simulated state as a passing runtime result.
